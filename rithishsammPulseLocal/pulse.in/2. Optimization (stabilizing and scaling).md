This document focuses on optimizing and stripping out what's already up and running without breaking the base and the bank.

> This optimization phase focuses on eliminating inefficiencies and aligning infrastructure with workload characteristics, ensuring cost-effective, high-performance operations without disrupting existing development velocity.

This phase focuses on improving the current infrastructure by:
- Reducing **cloud expenditure**
- Eliminating **inefficiencies**
- Aligning workloads with **appropriate environments (cloud vs on-prem)**
- Introducing **non-intrusive DevSecOps practices**
- Preparing for **controlled modernization**.

Objective is to reinvest these cost saving to take these workloads to a HA. 

With all this, the expect outcome to achieve
	1) Cost
		- Reduction of **~30–40% in cloud spend**
		- Elimination of ide/waste resources
		- Achieve Savings up to **~$8.5K+ USD/month (~₹6–7 Lakhs)** in cloud bills.  
	2) Performance
		- Improved latency (on-prem media handling)
		- Reduced network bottlenecks and cross-region traffic
	3) Security
		- Increased visibility into vulnerabilities
		- Gradual DevSecOps adoption
	4) Operations
		- Retain existing high-speed deployment workflows
		- Introduce **parallel feedback pipelines**
		- Improve system observability and operational control

#### Problem statements:
Fixing:
1) **Infrastructure Footprint**
	- Lack of resource segregation
	- No regional strategy
	- High networking complexity
	- Cost leakage via unused / scattered infra

2) **Operational Gaps**
	- No Infrastructure as Code (manual provisioning)
	- No standardized deployment model
	- Resource ownership unclear → stale resources
	- No audit / lifecycle governance
	- CI/CD exists but:
	    - No SecOps
	    - No feedback loops

3) **Cost & Performance Pressure**
- ~42% cost increase trend
- Major contributors:
    - EC2 (compute-heavy workloads)
    - RDS (persistent usage)
    - VPC/network transfer (voip traffic)
